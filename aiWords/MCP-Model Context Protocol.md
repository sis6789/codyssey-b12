Model Context Protocol (MCP)는 Anthropic이 개발하고 현재는 Linux Foundation 산하의 Agentic AI Foundation에서 관리하는 **AI 모델과 외부 세계(데이터, 도구)를 연결하는 오픈소스 개방형 표준 프로토콜**입니다.

기존 개발자들 사이에서는 흔히 "AI를 위한 USB-C 포트"라는 비유로 설명됩니다. 규격이 제각각이던 충전 단자가 USB-C로 통일된 것처럼, AI가 외부 데이터를 읽고 도구를 실행하는 방식을 하나로 통일하겠다는 취지입니다.

## 1. MCP가 등장한 이유 (N×M의 문제)

MCP가 나오기 전에는 AI 에이전트나 서비스(Claude, ChatGPT 등)에 특정 데이터(예: Notion, Slack, 기업 DB, 로컬 파일)를 연결하려면, **각 AI 모델별 + 각 데이터 소스별로 커스텀 연결 코드(커넥터)를 매번 새로 짜야 했습니다.**

AI 모델이 $N$개 있고 연결할 도구가 $M$개 있다면, 총 $N \times M$개의 커넥터를 만들어야 하는 엄청난 비효율이 발생했던 것이죠. MCP는 이를 중간에서 규격화(표준화)하여, 어떤 AI 모델이든 하나의 MCP 규격만 맞추면 수많은 외부 도구와 바로 연동될 수 있도록 해결했습니다.

## 2. 핵심 아키텍처 (어떻게 작동하나요?)

MCP는 크게 3가지 구성 요소로 움직이며, 주로 **JSON-RPC 2.0** 프로토콜 기반으로 통신합니다.

```
[ MCP Host (App / IDE) ] ── (내장) ──> [ MCP Client ] <─── JSON-RPC ───> [ MCP Server ] ──> (실제 데이터 / 도구)
```

- **MCP Host (호스트):** 사용자가 마주하는 AI 애플리케이션 환경입니다. (예: Claude Desktop, ChatGPT App, VS Code, Cursor 등)
    
- **MCP Client (클라이언트):** 호스트 앱 내부에서 AI 모델의 요청을 MCP 규격에 맞게 변환해 서버로 전달하고, 서버의 응답을 다시 모델이 이해할 수 있게 넘겨주는 연결통로입니다.
    
- **MCP Server (서버):** 실제 데이터 소스나 도구(컴파일러, 데이터베이스, 검색 엔진, 로컬 파일 시스템 등)의 기능을 노출하는 독립적인 서비스입니다.
    

## 3. MCP 서버가 제공하는 3가지 핵심 기능

MCP 서버는 클라이언트(AI)에게 크게 세 가지 형태의 능력을 공유합니다.

- **Resources (리소스):** AI가 읽을 수 있는 데이터(텍스트, 파일, API 응답, DB 레코드 등)를 제공합니다. 마치 웹의 URL처럼 AI가 데이터를 안전하게 가져올 수 있는 통로가 됩니다.
    
- **Tools (도구):** AI가 직접 실행할 수 있는 함수나 액션입니다. (예: "코드 실행하기", "DB에 가입자 정보 업데이트하기", "이메일 발송하기"). 사용자의 승인하에 AI가 능동적으로 연쇄 작업을 수행하게(Agentic behavior) 만듭니다.
    
- **Prompts (프롬프트):** 서버가 미리 정의해 둔 템플릿이나 워크플로우입니다. 사용자가 복잡하게 프롬프트를 짜지 않아도 서버가 제공하는 최적의 프롬프트 가이드를 AI가 바로 활용할 수 있습니다.
    

> 💡 **한 줄 요약**
> 
> MCP는 AI 모델이 로컬 컴퓨터의 파일, 회사 데이터베이스, 외부 API 도구들을 **하나의 통일된 규격으로 안전하게 읽고 쓸 수 있도록 만든 AI 업계의 새로운 표준 인터페이스**입니다. 현재 Anthropic뿐만 아니라 OpenAI, Google DeepMind, Microsoft 등 주요 기업과 수많은 개발자 도구(Cursor 등)가 이 생태계에 빠르게 동참하고 있습니다.


# ==외부에 공개되는 것은 무엇인가?==

MCP가 외부에 '오픈(Open)'하는 것은 한마디로 "컴퓨터 시스템의 제어권과 데이터에 접근할 수 있는 통일된 API 인터페이스"입니다.

조금 더 쉽게 비유하자면, 기존에는 AI가 모니터 너머로 대화만 나눌 수 있는 외딴섬에 갇혀 있었다면, MCP는 **AI가 내 컴퓨터나 서버 내부로 들어와서 합법적으로 일할 수 있도록 문을 열어주는 것**과 같습니다.

MCP 서버가 구체적으로 외부에 오픈하는 핵심 요소는 크게 3가지로 규격화되어 있습니다.

## MCP 서버가 외부에 오픈하는 3가지

### 1. Resources (데이터 및 컨텍스트 읽기 권한)

AI가 상황을 파악하고 답변을 생성하는 데 필요한 **로컬 데이터나 실시간 정보**를 오픈합니다.

- **오픈하는 것:** 로컬 파일(소스 코드, 로그, 마크다운 문서), 데이터베이스(SQL, 벡터 DB), 실시간 웹 검색 결과, Notion/Slack의 메시지 등.
    
- **효과:** AI가 단순히 학습된 지식으로만 답하는 게 아니라, 내 컴퓨터에 있는 파일이나 실시간 데이터를 직접 읽어와서 분석할 수 있게 됩니다.
    

### 2. Tools (행동 및 제어 권한)

AI가 수동적으로 읽기만 하는 것을 넘어, 실제 컴퓨터 시스템에 명령을 내리고 조작할 수 있는 '기능(함수)'을 오픈합니다.

- **오픈하는 것:** 터미널 명령어 실행기, 코드 컴파일러, 파일 수정/생성 기능, 외부 API 호출(이메일 발송, 브라우저 자동화 등).
    
- **효과:** AI에게 "이 코드 실행해서 에러 고쳐줘", "결과물 PDF로 저장해 줘"라고 명령하면 AI가 직접 도구를 실행해 업무를 완수합니다.
    

### 3. Prompts (최적화된 업무 가이드라인)

특정 데이터나 도구를 다룰 때 **AI가 어떻게 행동해야 하는지 정의된 워크플로우 템플릿**을 오픈합니다.

- **오픈하는 것:** "로그 분석을 위한 프롬프트 템플릿", "코드 리뷰 전용 가이드" 등.
    
- **효과:** 개발자가 복잡하게 명령어를 조합하지 않아도, MCP 서버가 미리 정의해 둔 최적의 프롬프트 양식을 AI가 즉시 가져다 쓰게 만듭니다.
    

## 🔒 오픈하되, 안전하게 제어합니다 (보안 아키텍처)

컴퓨터의 데이터와 제어권을 외부에 오픈한다고 하면 보안 우려가 생길 수밖에 없습니다. MCP는 이를 방지하기 위해 다음과 같은 방식으로 작동합니다.

- **샌드박스 기반 작동:** MCP 서버는 기본적으로 로컬(내 컴퓨터) 환경이나 지정된 보안 서버 안에서 실행됩니다. AI 모델(클라우드)에 내 컴퓨터 전체 제어권을 통째로 넘기는 것이 아니라, **내가 실행한 MCP 서버가 허용한 범위(특정 폴더, 특정 API 명령) 안에서만 AI가 요청**을 보낼 수 있습니다.
    
- **사용자 승인(Human-in-the-loop):** AI가 오픈된 `Tool`을 사용해 치명적인 작업(예: 파일 삭제, 코드 실행, 이메일 전송)을 하려고 할 때, 호스트 앱(예: Claude Desktop, VS Code)이 사용자에게 "이 도구를 실행할까요?"라는 팝업을 띄워 최종 승인을 받도록 설계되어 있습니다.
    

> **결론적으로**
> 
> MCP는 **내 컴퓨터의 파일(Resources)과 실행 기능(Tools)**을 AI가 이해할 수 있는 표준 규격으로 외부에 오픈하여, AI를 단순한 '채팅 로봇'에서 내 컴퓨터를 대신 움직여주는 **'자율형 에이전트(Agent)'**로 진화시키는 열쇠입니다.


# ==서버와 AI==

"MCP 서버를 AI가 읽고 조작한다"고 할 때, 여기서 지칭하는 **AI**는 단순히 하나의 두뇌(인공지능 모델)만을 뜻하는 것이 아닙니다.

MCP 아키텍처에서 AI는 ‘두뇌(LLM)’와 그 두뇌가 탑재되어 사용자와 소통하는 ‘애플리케이션(Host/Client)’이 결합된 시스템 전체를 의미합니다. 구체적으로 나누어 보면 다음과 같습니다.

## 1. 두뇌 역할: 대형 언어 모델 (LLM)

컴퓨터의 생각을 담당하는 진짜 인공지능 모델입니다.

- **대상:** OpenAI의 **GPT-4o**, Anthropic의 **Claude 3.5 Sonnet**, Google의 **Gemini 1.5 Pro** 같은 모델들입니다.
    
- **하는 일:** 사용자가 대화창에 명령을 내리면, 자신의 지식만으로는 해결할 수 없다는 것을 인지하고 "**구글 캘린더 MCP 서버**에 있는 `create_event` 도구를 실행해 줘"라고 **판단하고 판단 결과를 구조화된 데이터(JSON)로 출력**합니다.
    

## 2. 손발 역할: AI 애플리케이션 및 IDE (MCP Host / Client)

사용자가 실제로 마주하고 타자를 치는 프로그램입니다. 모델이 내린 판단을 받아 **실제 MCP 서버에 신호를 보내고 조작을 대행하는 손발**이 됩니다.

- **대상:**
    
    - **AI 특화 개발 도구 (IDE):** Cursor, VS Code, Windsurf 등
        
    - **AI 채팅 앱:** Claude Desktop, ChatGPT App 등
        
    - **자율형 에이전트 프레임워크:** LangChain, AutoGPT 기반 앱 등
        
- **하는 일:** 두뇌(LLM)가 "구글 드라이브 파일 읽어줘"라고 요청하면, 이 앱 내부에 심어진 **MCP 클라이언트**가 구글 MCP 서버에 접속해 파일을 읽어와 다시 두뇌에게 전달해 줍니다.
    

## 🔄 실제 작동 예시로 보기

사용자가 **Cursor(AI 개발 프로그램)** 창에 이렇게 입력했다고 가정해 보겠습니다.

> _"내 구글 드라이브에 있는 `schema.txt` 파일을 읽고, 여기에 맞춰 데이터베이스 테이블을 만들어줘."_

이때 'AI'는 다음 과정으로 MCP 서버를 읽고 조작합니다.

```
[ 사용자 ] ── "구글 드라이브 읽어줘" ──> [ Cursor (호스트 앱) ]
                                                │ (질문 전달)
                                                ▼
                                         [ Claude/Gemini (두뇌) ]
                                                │ "구글 MCP 서버의 '파일 읽기' 도구 써줘" (판단)
                                                ▼
[ 구글 Drive MCP 서버 ] <── (명령 실행) ── [ Cursor 내 MCP 클라이언트 ]
```

1. **두뇌 (Claude/Gemini):** 사용자의 말을 분석한 뒤, 본인에게 구글 드라이브 접근 권한이 없으므로 연동된 구글 MCP 서버의 `read_file`이라는 도구를 호출하라는 명령(JSON)을 내립니다.
    
2. **손발 (Cursor 앱):** 두뇌의 명령을 받아 구글 MCP 서버에 안전하게 접속해 `schema.txt` 내용을 **읽어옵니다(Read)**.
    
3. **두뇌 (Claude/Gemini):** 읽어온 텍스트를 보고 SQL 데이터베이스 생성 코드를 짭니다.
    
4. **손발 (Cursor 앱):** 다시 로컬 DB MCP 서버를 이용해 사용자의 컴퓨터에 테이블을 **생성합니다(Write/조작)**.
    

> **결론**
> 
> 여기서 말하는 AI는 **"Claude나 Gemini 같은 최신 LLM(두뇌)"**과 **"이를 움직이는 Cursor, Claude Desktop 같은 AI 서비스(손발)"**가 합쳐진 통합 에이전트 시스템을 뜻합니다. MCP 서버는 이 통합 AI 시스템이 외부 세상을 안전하게 탐색할 수 있도록 열어둔 전용 인터페이스입니다.
